CLI (command line inter face)

명령줄 인터페이스
  • ls => 를 다보여줘라
  • touch cli.txt => cli.txt를 생성해라
  • mkdir git => git 디렉토리를 제작해라
  • cd git => git 디렉토리로 가라
  • cd .. => 이전 디렉토리로 가라
  • git init => git을 시작한다. (저장소 생성)
  • git add . => commint할 목록에 등록
  • git commit -m ‘update files’ => 커밋 수행
  • git log => 버전 확인
  • git status => 상태 확인

Git 기초

Git은 분산형 버전관리시스템(DVCS)이다.

Git을 윈도우에서 활용하기 위해서 git bash를 설치해야 한다.

1. 저장소 초기화

$ git init
Initialized empty Git repository in C:/Users/i/Desktop/TIL/.git/

(master) $
  • 로컬 저장소를 만들고 나면, git/폴더가 생성되고, bash에 (master)라고 표기된다.
  • 반드시 저장소를 만들기 전에 원하는 디렉토리인지 확인하는 습관을 가지고, 저장소 내부에 저장소를 만들지 말자.
    • 예) Desktop -> git 저장소, TIL->다른 git 저장소(x)

2. add

작업한 내용을 커밋대상 목록에 추가한다.

$ git add .               	# 현재 디렉토리(하위 디렉토리 포함)
$ git add a.html			# 특정 파일
$ git add b.html c.html		# 특정 다수 파일
$ git add blog/				# 특정 폴더

$ git status
On branch master

No commits yet
# untracked files => git으로 관리된 적 없는 파일
Untracked files:
# 커밋 될 것들에 포함시키기 위해서는 add 명령어를 써라

3. Commit

$ git commit -m 'Add markdown.md'
[master (root-commit) fc4422c] Add markdown.md
 3 files changed, 89 insertions(+)
 create mode 100644 markdown-images/capture-1597899896883.PNG
 create mode 100644 markdown-images/capture.PNG
 create mode 100644 markdown.md
  • 커밋은 버전(이력)을 기록하는 명령어이다.
  • 커밋 메시지는 해당하는 이력을 나타낼 수 있도록 작성 하여야 한다.
  • 커밋 이력을 확인하기 위해서는 아래의 명령어를 사용한다.
$ git log

Git 원격 저장소 활용

Git 원격 저장소 기능을 제공 해주는 서비스는 gitlab,bitbucket,github등이 있다.

0. 원격 저장소 설정

$ git remote add origin {url}
$ git remote add origin https://github.com/heohyoyeong/TIL.git

  • git, 원격저장소를 추가 (add)하고 origin이라는 이름으로 url을 설정

    git아 원격저장소를 추가해줘 origin이라는 이름으로 url을

1. 설정된 저장소를 확인하기 위해서는 아래의 명령어를 사용한다.

$ git remote -v
