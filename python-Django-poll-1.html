이전 포스트에서 제작하였던 polls 폴더에 존재하였던 manage.py를 수정합니다.
수정을 수행할 때 파이썬은 객체지향이기 때문에 class를 사용합니다.
또한 질문과 보기를 따로 나타내야하기때문에 2개의 class를 제작할것입니다.

Question class 제작

  • 투표의 질문을 작성할 class를 제작합니다.
  • 예를 들어. 가장 좋아하는 숫자는? 가장 좋아하는 연예인은?과 같은 질문들이 들어갈 수 있는 class입니다.
class Question(models.Model):
    question_text = models.CharField(max_length=200) #charfield는 문자열을 지칭 max_length는 200글자까지만 들어간다!
    pub_date=models.DateTimeField("date published")  #DatetimeField는 날짜가 들어감 "date published" 설명(이건 안들어가도됨)
    # Intgerfield는 정수열이 들어감
    # table의 id column은 default로 지정된다.
    # id가 primary key, Not Null, unique
    # id는 autoincrement 특성을 가지는 정수형으로 지정.
    # 자동을 생성해주기 떄문에 class정의에서는 나오지 않는다!
    # def __repr__(self):  # repr은 객체를 임의로 특정 문자열로 표현하고 싶을떄 사용

    def __str__(self): # str은 연산을 위하여 문자열로 바꾸고 싶을때
        return self.question_text
  • 위와 같이 Question class를 제작합니다.

Choice class 제작

  • 투표의 내용을 작성할 class를 제작합니다.
  • 예를 들어. 가장 좋아하는 숫자는? 라면 1,2,3,4가 들어갈 수 있습니다.
  • 가장 좋아하는 연예인은?과 같은 질문들은 송지효, 전소민과 같은 것들이 들어갈 수 있는 class입니다.
class Choice(models.Model):
    question=models.ForeignKey(Question,on_delete=models.CASCADE)
    # question의 primary key를 가져오기때뭉네 foreignkey로 지정하고 question을 받는다.
    # on_delete의 경우 만약 question의 id가 삭제되면 그 id와 관련된 값들을 삭제 해버린다!
    # question_id는 foreignkey이기 때문에 자동으로 _id가 추가되어 question이라고만 작성한다.
    choice_text=models.CharField(max_length=200)
    votes = models.IntegerField(default=0)
    #integerfield의 default를 0으로 하면 초기값이 0으로 설정

    def __str__(self):
        return self.choice_text
  • class가 완성되었다면 admin.py를 수정하여 관리할 수 있게 합니다.

admin.py 수정

  • 아래와 같이 admin.py를 수정합니다.
from django.contrib import admin

#models.py에서 작성한 Question과 Choice를 불러온다!
from polls.models import Question
from polls.models import Choice

#괄호안에는 내가 등록할 클래스가 나와야 한다.
admin.site.register(Question)
admin.site.register(Choice)
  • 데이터베이스에 테이블을 만들려면 SQL구문이 필요합니다.
  • 이를 위해 C:\Djangotest2/MyFirstWebPoll에서 python manage.py makemigrations과 python manage.py migrate를 수행합니다.
  • 마지막으로 이전에 확인하였던 admin 싸이트를 확인해보면 아래와 같이 변경된것을 확인할수있습니다.
