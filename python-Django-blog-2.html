이번 포스트에서는 작성한 User model을 사용하여 회원가입과 로그인을 구현해보려고합니다.

가장먼저 초기화면의 url부터 수정하여 초기화면에서 로그인과 회원가입을 할수있게 하려고합니다.
초기화면을 수정해주기 위하여 가장먼저 project의 settings.py 부터 수정해주겠습니다.

setting 수정

  • 이전의 투표 프로젝트와 유사하게 INSTALLED_APPS에 ‘{application_name}.apps.{application_name}Config’을 추가해주겠습니다.
  • 하지만 이전과는 다르게 html을 bootstrap을 이용하여 추가 수정해줄것이기때문에 ‘bootstrap4’도 추가해주겠습니다.
  • 그후 초기화면의 templates를 추가해주기위하여 TEMPLATES에 ‘DIRS’: [os.path.join(BASE_DIR,’{project_name}’,’templates’)],을 추가해줍니다.
  • 마지막으로 가장 마지막줄인 STATIC_URL의 밑에 STATICFILES_DIRS = [os.path.join(BASE_DIR,’static’)]을 추가해줍니다.

url 수정

  • 가장 먼저 project의 url을 수정해주겠습니다.
  • 과거 투표프로젝트에서는 기초화면은 건들이지 않았지만 블로그에서는 수정을 해줄것입니다.
  • 아래의 url pattern에서 r”^$”는 정규식 표현입니다.
  • r은 \를 한개만써주겟다는 의미이고 ““내부의 ^는 시작 $는 끝을 의미합니다.
  • 결과적으로 ^와 $사이에 아무것도 없으므로 초기화면을 의미합니다.
  • 나머지 admin과 posts는 투표프로젝트에서 사용하였던것과 동일합니다.
from django.contrib import admin
from django.urls import path, include 
from django.conf.urls import url # 이렇게 해야 url을 사용할수있다.
from django.views.generic.base import TemplateView

urlpatterns = [
    url(r"^$", TemplateView.as_view(template_name="index.html"), name="home"),  
    # http://localhost:8000일때 view의 함수를 거치지않고 index.html을 가져와! 라는 의미이다.
    path('admin/', admin.site.urls),
    path('posts/', include('posts.urls'))
]

        res_data = {}

            res_data['error'] = "모든 값을 입력해야 합니다."

            return HttpResponse('비밀번호가 다릅니다.')

def login(request):

        return render(request, 'login.html')

    elif request.method == "POST":
        login_username = request.POST.get('username', None)
        login_password = request.POST.get('password', None)


        my_list = Post.objects.all().order_by("-id")
